Reflection

What went well:

This was one of my most robust projects, because I went through all the steps in UX and UI design. For example, conducting UX research to understand user needs, helped in creating user personas, MVP, jobs to be done, and user flow diagrams. Once I had the research and a general direction for my project, it made it easier to go into the UI process of creating the wireframes and style guides. It was also helpful to learn how to conduct user tests and user preference tests to further refine my designs by getting real feedback from potential users. Overall I think it was helpful to go through the whole process, and learn how to create each of the components from start to finish.

What could be improved:

This project was where I first learned about the UX and UI design process and how to create different deliverables at each step; so I think there was a learning curve by having to learn and practice these new skills simulatneously. Therefore, there were many revisions that occurred throughout this project, from the wireframes, layout, and style. I believe deigns can always be improved, and for this project specifically I think there is room for improvement in the spacing, layout, and funcitonality. This was also my first project where I had to design for different breakpoints (mobile, tablet, desktop), so I think I can improve the tablet and desktop screens and continue to practice my skills in designing for different breakpoints.
